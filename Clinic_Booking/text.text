المرحلة الأولى: Dashboard
الخطوة 1: التأسيس

إنشاء مشروع Clinic_Admin.
إعداد Layout، القائمة الجانبية، Header، تسجيل الدخول.
ربط Axios وJWT وحماية Routes حسب الدور.
تجهيز مكونات مشتركة: الجداول، Pagination، النوافذ، رسائل النجاح والخطأ، رفع الصور.
الخطوة 2: لوحة المدير SuperAdmin

صفحة إحصائيات أولية.
إدارة المستخدمين: عرض، بحث، قفل، حذف.
إدارة الأطباء: إضافة، تعديل، حذف، ربط الحساب، الموافقة على الظهور.
عرض صفحة الطبيب وعياداته.
الخطوة 3: إدارة الاشتراكات

عرض الباقات والمميزات.
إنشاء اشتراك للطبيب.
تفعيل، تجديد، ترقية وإلغاء الاشتراك.
إظهار حالة الاشتراك وتاريخ الانتهاء.
الخطوة 4: حساب الطبيب DoctorUser

تعديل الملف الشخصي والصورة.
إدارة العيادات المتعددة.
إدخال العنوان ورابط الخارطة والإحداثيات.
إدارة دوام كل عيادة وحد الأدوار اليومية.
الخطوة 5: الحجوزات والإجازات

عرض حجوزات العيادة حسب التاريخ والحالة.
تغيير حالة الحجز وإكماله.
إدارة الإجازات والاستثناءات.
إغلاق يوم أو تعديل عدد الأدوار لذلك اليوم.
الخطوة 6: التقييمات والتحسين النهائي

عرض تقييمات الطبيب ومتوسطها.
تحسين تجربة الاستخدام والتحميل والأخطاء.
اختبار الصلاحيات والتنقل.
اختبار Responsive.
المرحلة الثانية: تطبيق المستخدم
الخطوة 1: التأسيس والتصميم

إنشاء مشروع Clinic_App.
Navbar وFooter وتصميم RTL متجاوب.
إعداد Axios وPinia وRouter.
صفحات تسجيل الحساب والدخول واستعادة كلمة المرور وتأكيد البريد.
الخطوة 2: البحث والدليل الطبي

الصفحة الرئيسية.
البحث حسب المحافظة والاختصاص واسم الطبيب.
عرض قائمة الأطباء مع الصورة والتخصص والتقييم.
صفحة الطبيب: الوصف، العيادات، العنوان، الخارطة، الهاتف والدوام.
الخطوة 3: الحجز بالدور

اختيار العيادة.
عرض الأيام المتاحة والأدوار المتبقية.
اختيار التاريخ فقط.
الحجز بحساب أو كزائر.
عرض رقم الدور وكود الحجز.
شاشة OTP عند تفعيل الإعداد.
الخطوة 4: إدارة الحجوزات

صفحة حجوزاتي للمستخدم المسجل.
مراجعة الحجز وإلغاؤه.
صفحة مستقلة للزائر باستخدام الهاتف وكود الحجز.
رسائل واضحة للحالات: مؤكد، انتظار تحقق، مكتمل، ملغي.
الخطوة 5: التقييمات

إضافة تقييم بعد اكتمال الحجز.
عرض متوسط التقييم وعدد التقييمات.
إخفاء القسم تلقائياً إذا كانت باقة الطبيب لا تسمح به.
الخطوة 6: الإطلاق

حالات التحميل وEmpty States.
تحسين الموبايل.
معالجة أخطاء API.
اختبار تدفق المستخدم والزائر من البحث حتى الإلغاء والتقييم.
إعداد ملفات البيئة للإنتاج.
