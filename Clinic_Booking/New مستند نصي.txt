المرحلة 1: إعادة تنظيم قاعدة البيانات

إنشاء جدول Clinics للعيادات:
اسم العيادة.
الطبيب المالك.
المحافظة.
العنوان الكامل.
رابط الموقع على الخارطة أو إحداثيات Latitude وLongitude.
رقم هاتف العيادة.
حالة الظهور والحذف.
نقل الدوام الأسبوعي من الطبيب إلى العيادة:
كل فرع يملك أيام دوام مستقلة.
بداية ونهاية الدوام تبقى لأغراض العرض فقط.
تحديد الحد الأعلى للأدوار اليومية.
تعديل الحجز ليرتبط بالعيادة والطبيب معاً.
جعل UserId اختيارياً حتى يقبل النظام الحجز كزائر.
إضافة بيانات الزائر إلى الحجز:
الاسم.
رقم الهاتف.
ملاحظات اختيارية.
إضافة رقم دور QueueNumber لكل حجز.
إضافة سبب الإلغاء ومن قام بالإلغاء.
إضافة حالة تأكيد الهاتف للحجز.
المرحلة 2: نظام الحجز بالدور

المستخدم يختار المحافظة ثم الاختصاص ثم الطبيب ثم العيادة.
تظهر له أيام دوام العيادة وعدد الأدوار المتبقية لكل يوم.
يختار التاريخ فقط، بدون اختيار وقت.
النظام يولد تلقائياً:
رقم الدور.
كود حجز فريد.
منع الحجز إذا:
العيادة مغلقة في هذا اليوم.
الاشتراك لا يسمح بالحجز الإلكتروني.
العدد اليومي وصل للحد الأعلى.
التاريخ قديم.
المستخدم أو الزائر لديه حجز فعال مكرر بنفس العيادة واليوم.
حماية العملية من إصدار نفس الدور لشخصين عند وصول طلبين بنفس اللحظة.
المرحلة 3: الحجز بحساب أو كزائر

الحجز بحساب:
تُملأ معلومات المستخدم تلقائياً.
يستطيع مشاهدة حجوزاته وإلغاءها.
الحجز كزائر:
يدخل الاسم ورقم الهاتف.
يستلم كود الحجز.
يستطيع مراجعة الحجز أو إلغاءه باستخدام الهاتف والكود.
توفير API للاستعلام عن الحجز العام بدون كشف بيانات حساسة.
المرحلة 4: OTP اختياري

إضافة إعداد عام مثل EnableBookingOtp.
عند تعطيله:
ينشأ الحجز مباشرة.
عند تفعيله:
ينشأ الحجز بحالة انتظار التحقق.
يُرسل رمز إلى رقم الهاتف.
لا يثبت الدور إلا بعد إدخال الرمز خلال مدة محددة.
إضافة جدول OtpRequests أو حقول مناسبة لتخزين:
الرمز بشكل مشفر.
تاريخ الانتهاء.
عدد المحاولات.
وقت آخر إرسال.
تجهيز واجهة إرسال OTP بشكل قابل لربط مزود رسائل لاحقاً، بدون إلزامك بمزود الآن.
المرحلة 5: حساب الطبيب وإدارة عياداته

ربط سجل الطبيب بحساب مستخدم يحمل دور DoctorUser.
السماح للطبيب بإدارة:
بياناته الشخصية.
عياداته المتعددة.
عناوين العيادات ومواقعها على الخارطة.
الدوام الأسبوعي لكل عيادة.
إغلاق يوم محدد أو إضافة إجازة.
الحجوزات اليومية وتغيير حالتها.
إضافة موظفي عيادة اختيارياً لاحقاً، حتى يدير موظف الاستقبال الأدوار بدون امتلاك صلاحيات الطبيب كاملة.
المرحلة 6: البحث وصفحة الطبيب

تطوير البحث حسب:
المحافظة.
الاختصاص.
اسم الطبيب.
صفحة الطبيب تعرض:
الصورة والوصف والتخصص.
العيادات التابعة له.
عنوان كل عيادة.
الموقع على الخارطة.
رقم الهاتف.
أيام الدوام.
إمكانية الحجز حسب الاشتراك.
التقييمات إذا كانت الميزة مفعلة.
إضافة حالة ظهور الطبيب للعامة بعد موافقة الإدارة.
المرحلة 7: الاشتراكات والمميزات

إبقاء الباقات الحالية مع تنظيمها.
ربط الاشتراك بالطبيب، وليس بعيادة منفردة مبدئياً.
تطبيق حدود الباقة فعلياً:
عدد العيادات المسموح بها.
عدد أيام الدوام الأسبوعية لكل عيادة أو للطبيب ككل.
أقصى عدد أدوار يومية.
الحجز الإلكتروني.
التقييمات.
الرسائل.
الدفع الإلكتروني.
العروض.
إضافة حالات الاشتراك:
Pending
Active
Expired
Cancelled
دعم التجديد والترقية والإلغاء.
تعطيل المميزات تلقائياً عند انتهاء الاشتراك.
تحديد سياسة الطبيب بدون اشتراك: المقترح أن يبقى ظاهراً كدليل، لكن بدون زر حجز إلكتروني.
المرحلة 8: الإجازات والاستثناءات

إنشاء جدول ClinicExceptions.
دعم الحالات:
العيادة مغلقة في تاريخ معين.
تغيير عدد الأدوار ليوم معين.
تعديل وقت العرض لذلك اليوم.
عرض سبب الإغلاق للمستخدم عند الحاجة.
المرحلة 9: التقييمات

إكمال API الخاص بالتقييمات.
السماح بالتقييم فقط بعد اكتمال حجز حقيقي.
منع تكرار التقييم لنفس الحجز.
عرض متوسط التقييم وعدد التقييمات.
إخفاء التقييمات إذا كانت باقة الطبيب لا تسمح بها.
المرحلة 10: الحماية والصلاحيات

تفعيل المصادقة بشكل صحيح وإضافة UseAuthentication().
حماية الواجهات باستخدام [Authorize].
فصل الصلاحيات:
SuperAdmin
DoctorUser
ClinicStaff
NormalUser
زائر بدون حساب
منع المستخدم من تعديل أو رؤية بيانات لا تخصه.
إزالة كلمات المرور ومفاتيح JWT والبريد من الملفات المرفوعة إلى Git.
إزالة حساب المدير الافتراضي ذي كلمة المرور الثابتة أو إنشاءه بطريقة آمنة.

إضافة endpoint لتعديل الطبيب.
إكمال تسجيل خدمة التقييم وإضافة Controller.
إصلاح حذف الصورة القديمة للمستخدم.
إصلاح روابط تأكيد البريد واسترجاع كلمة المرور.
تعديل اسم Swagger من Blood Bank إلى اسم المشروع.
تحسين رسائل الأخطاء والتحقق من المدخلات.
إضافة فهارس قاعدة بيانات للحجوزات والبحث.
إضافة اختبارات للحجز والاشتراكات والصلاحيات.

إشعارات SMS أو WhatsApp.

الإحالات بين الأطباء.

قائمة المفضلة.
