# التحليل الشامل للنظام

تاريخ التحليل: 2026-06-12  
النطاق: الباك إند `Clinic_Booking`، واجهة Vue `Clinic_Admin`، تطبيق Flutter `Clinic_App`.

## 1. نظرة عامة

النظام عبارة عن منصة حجز عيادات تشمل ثلاثة أسطح تشغيل:

- `Clinic_Booking`: API مبني بـ ASP.NET Core و Entity Framework Core مع JWT، أدوار، خدمات خلفية، إشعارات Firebase، WhatsApp OTP، وإدارة حجوزات/أطباء/عيادات/اشتراكات.
- `Clinic_Admin`: لوحة Vue/Vite للإدارة والطبيب، وفيها أيضاً صفحات عامة للدليل والحجز كزائر.
- `Clinic_App`: تطبيق Flutter للمريض والطبيب، يدعم الدليل، الحجز، OTP، الحجوزات، حساب الطبيب، العيادات، الدوام، العروض، الإشعارات، والمفضلة.

الأدوار الأساسية:

- `SuperAdmin`: إدارة المستخدمين، الأطباء، الاشتراكات، الإصدارات، WhatsApp OTP، الإحصائيات، العروض، ومراقبة الحجوزات.
- `DoctorUser`: إدارة ملف الطبيب، العيادات، الدوام، الاستثناءات، الحجوزات، العروض، التقييمات، الإشعارات، والاشتراك.
- المستخدم/المريض: التسجيل، الدخول، البحث عن الأطباء، الحجز، متابعة الحجوزات، التقييم، المفضلة.
- الزائر: حجز غير مسجل عبر OTP ومتابعة الحجز بالكود ورقم الهاتف.

## 2. الباك إند

### 2.1 البنية العامة

المشروع يستخدم:

- Controllers لكل نطاق: `User`, `Doctor`, `Clinic`, `Appointment`, `DoctorAvailability`, `ClinicException`, `DoctorSubscription`, `DoctorOffer`, `Review`, `Analytics`, `AppVersion`, `FavoriteDoctor`, `Notification`.
- Services لكل منطق أعمال رئيسي.
- `ApplicationDbContext` يحتوي على الجداول الأساسية والفهارس.
- Hosted services:
  - `SubscriptionExpirationService`: متابعة انتهاء الاشتراكات والتنبيهات.
  - `AppointmentReminderService`: تذكير الحجوزات.
  - `DataCleanupService`: تنظيف OTP والإشعارات المقروءة.

### 2.2 الكيانات والجداول

الجداول الرئيسية:

- المستخدمون والأدوار: `AspNetUsers`, `AspNetRoles`.
- الأطباء: `Doctors`, `Specializations`.
- العيادات والدوام: `Clinics`, `DoctorAvailabilities`, `Days`, `ClinicExceptions`.
- الحجوزات: `Appointments`, `BookingOtpRequests`.
- الاشتراكات والميزات: `SubscriptionPackages`, `DoctorSubscriptions`, `Features`, `DoctorFeature`.
- العروض: `DoctorOffers`.
- التقييمات: `Reviews`.
- الإشعارات: `Notifications`, `DeviceTokens`.
- الإحصائيات: `AnalyticsEvents`.
- المفضلة: `UserFavoriteDoctors`.
- OTP الهاتف: `UserPhoneOtpRequests`.
- الإصدارات: `AppVersionPolicies`.

تمت إضافة حقول/جداول حديثة:

- `Clinics.BookingWindowDays`: عدد الأيام المتاحة للحجز المسبق، الافتراضي 7.
- `Notifications.ReadAt`: وقت قراءة الإشعار.
- `UserFavoriteDoctors`: علاقة المريض مع الأطباء المفضلين.

### 2.3 المصادقة والصلاحيات

المصادقة عبر JWT و refresh tokens. توجد حماية بالأدوار على أغلب مسارات الإدارة والطبيب. المسارات العامة تشمل دليل الأطباء، توفر الحجز، الحجز كزائر، OTP، متابعة حجز الزائر، والإصدارات.

نقاط جيدة:

- فصل واضح بين صلاحيات `SuperAdmin` و `DoctorUser`.
- تحديث refresh token.
- تسجيل device token لإشعارات Firebase.

نقاط تحتاج ضبط تشغيلي:

- روابط البريد/استرجاع كلمة المرور في `UserServices` تحتوي `http://localhost:5173` بشكل ثابت.
- إعدادات Firebase و WhatsApp تعتمد على ملفات/قيم بيئة ويجب فصلها عن المستودع.

### 2.4 سيناريو الحجز

التدفق الحالي:

1. المريض أو الزائر يطلب توفر الأدوار عبر `/Appointment/queue-availability/{clinicId}`.
2. النظام يفحص جدول الدوام الأسبوعي والاستثناءات وعدد الحجوزات الحالية.
3. عند الحجز:
   - للمستخدم المسجل: ينشأ حجز ويطلب تأكيد الطبيب أو OTP حسب السيناريو.
   - للزائر: ينشأ حجز مع OTP ويرتبط برقم الهاتف والكود.
   - للحجز اليدوي من الطبيب: ينشأ حجز مؤكد/نشط حسب منطق الخدمة.
4. الطبيب يستطيع تأكيد/إلغاء/إكمال الحجز.
5. المريض يستطيع إلغاء حجزه من التطبيق أو الزائر من صفحة متابعة الحجز إذا كان الإلغاء مسموحاً.

عناصر الحماية:

- فهرس فريد على `ClinicId + AppointmentDate + QueueNumber` لتقليل تضارب الأدوار.
- إعادة محاولة عند تعارض رقم الدور.
- انتهاء OTP يلغي الحجز غير المؤكد.

### 2.5 الدوام والاستثناءات

يدعم النظام:

- جدول دوام أسبوعي لكل عيادة.
- تعديل يوم واحد من جدول الدوام.
- استثناء تاريخ محدد:
  - إغلاق كامل.
  - دوام مخصص بعدد أدوار/وقت.
  - عند الإغلاق يمكن نقل الحجوزات أو إلغاؤها.

التحديثات الأخيرة تضيف:

- عند إضافة يوم استثناء مغلق ومع وجود حجوزات:
  - `appointmentConflictAction = move`: نقل الحجوزات إلى تاريخ جديد.
  - `appointmentConflictAction = cancel`: إلغاء الحجوزات.
  - إرسال إشعارات للمرضى المسجلين وحفظ إشعار للطبيب.
- عند تعطيل يوم دوام أسبوعي: نقل الحجوزات المستقبلية إلى أول يوم دوام متاح وإشعار المرضى.

### 2.6 الاشتراكات

النظام يدعم:

- باقات اشتراك وميزات مرتبطة بها.
- إنشاء/تفعيل/تجديد/ترقية/إلغاء اشتراك من لوحة المدير.
- اشتراك الطبيب الحالي `/DoctorSubscription/my/current`.
- خدمة خلفية لانتهاء الاشتراك:
  - إرسال تنبيهات قبل 3 و2 و1 يوم.
  - تحويل الاشتراك المنتهي إلى منتهي.
  - إعادة الطبيب للخطة الأساسية إذا لم توجد باقة فعالة أخرى.
  - تعطيل الميزات المدفوعة وإعادة قيود الخطة المجانية.

### 2.7 الإشعارات

يوجد مساران:

- Push notifications عبر Firebase إلى المستخدمين الذين لديهم device tokens.
- جدول `Notifications` لإشعارات الطبيب مع القراءة والحذف الآلي بعد 30 يوم من القراءة.

توجد شاشة للطبيب في Flutter و Vue لعرض إشعاراته وتعليمها كمقروءة.

### 2.8 التحليلات

يوجد `AnalyticsEvent` ومسارات:

- تتبع حدث عام.
- ملخص مدير النظام.
- ملخص الطبيب.
- ملخص طبيب محدد للمدير.

الواجهة تعرض مؤشرات حجوزات، مشاهدات، ضغطات الحجز، عروض، نمو، وأحداث حديثة.

## 3. واجهة Vue

### 3.1 التقنية والبنية

الواجهة مبنية بـ Vue 3 + Vite + Pinia + Vue Router + Axios.  
المسارات مقسمة إلى:

- عامة: login، تأكيد البريد، إعادة كلمة المرور، الدليل العام، صفحة الطبيب العامة، متابعة حجز زائر.
- داخل `AdminLayout`: dashboard، analytics، users، doctors، subscriptions، app versions، whatsapp، offers، clinics، appointments، exceptions، notifications، reviews، profile.

### 3.2 وظائف المدير

يدعم المدير:

- إدارة المستخدمين.
- إدارة الأطباء وربط حساب طبيب.
- إدارة الاشتراكات والباقات والميزات.
- إدارة إصدارات التطبيق.
- إدارة WhatsApp OTP.
- مراقبة الحجوزات.
- الإحصائيات.
- العروض.

### 3.3 وظائف الطبيب

يدعم الطبيب:

- لوحة رئيسية مع عداد الاشتراك.
- تغيير ثيم الواجهة إلى ذهبي عند الباقة المميزة.
- عياداتي: بيانات العيادة، سعر المراجعة، الموقع، الرؤية، وعدد أيام الحجز المسبق.
- الدوام الأسبوعي لكل عيادة.
- الاستثناءات مع خيار نقل/إلغاء الحجوزات عند الإغلاق.
- الحجوزات اليومية مع فلتر تاريخ اليوم افتراضياً.
- زر اتصال مباشر بالمريض.
- إشعارات الطبيب.
- العروض.
- التقييمات.
- الملف الشخصي.

### 3.4 الواجهة العامة في Vue

تدعم:

- دليل الأطباء العام.
- صفحة طبيب عامة.
- الحجز كزائر مع OTP.
- متابعة حجز الزائر وإلغاؤه.

تم ربط `bookingWindowDays` في صفحة الطبيب العامة بحيث يحدد الحد الأعلى لتاريخ الحجز في الواجهة.

### 3.5 حماية العمليات الحساسة

تمت إضافة `LongPressButton` واستخدامه في:

- حذف عيادة.
- حذف استثناء.
- حذف عرض.
- إلغاء حجز.
- إلغاء اشتراك.

## 5. التحديثات الـ 13 وحالتها

1. إضافة يوم استثناء مع معالجة الحجوزات: منفذ في الباك، Vue، Flutter.

5. تغيير هوية التطبيق حسب الاشتراك: منفذ في Flutter وVue للطبيب المميز.
6. تحديد عدد أيام الحجز لكل عيادة: منفذ في الباك وFlutter وVue، ومستخدم في عرض التوفر.
7. نظام إشعارات الطبيب: منفذ في الباك وFlutter وVue.
8. حذف OTP المنتهي: منفذ عبر `DataCleanupService`.
9. تعطيل يوم دوام مع نقل الحجوزات: منفذ في الباك مع تنبيه في الواجهات.
10. انتهاء الاشتراك والعودة للخطة المجانية: منفذ في خدمة خلفية.
11. تنبيهات قرب انتهاء الاشتراك: منفذة في خدمة خلفية.
12. عداد الأيام المتبقية: منفذ في Flutter وVue.
