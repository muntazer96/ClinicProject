خطوات تحديث مشروع العيادة على السيرفر

استخدم هذا الملف كل مرة ترفع تحديث جديد على GitHub وتريد تحدث السيرفر.

مسار المشروع /opt/clinic/ClinicProject
البرانج message-version
عنوان السيرفر 167.86.119.12

1. الدخول للسيرفر

ادخل إلى السيرفر عن طريق SSH:

ssh root@167.86.119.12

2. الانتقال إلى المشروع

cd /opt/clinic/ClinicProject

3. التأكد من البرانج الصحيح

تأكد أنك على برانج message-version:

git branch

إذا لم تكن عليه، نفذ:

git checkout message-version

4. سحب آخر تحديث من GitHub

git pull origin message-version
إذا ظهر تعارض بسبب ملفات السيرفر مثل .env.production أو package-lock.json، لا تستخدم git reset مباشرة. انسخ الخطأ وراجعه قبل المتابعة.

5. تحديث الباكند .NET

cd /opt/clinic/ClinicProject/Clinic_Booking
dotnet restore
dotnet publish -c Release -o /var/www/clinic-api
systemctl restart clinic-api

تأكد أن الخدمة اشتغلت بدون مشاكل:

systemctl status clinic-api

6. تحديث الفرونت Vue

cd /opt/clinic/ClinicProject/Clinic_Admin
npm install
npm run build
rm -rf /var/www/clinic-admin/*
cp -r dist/* /var/www/clinic-admin/
chown -R www-data:www-data /var/www/clinic-admin
systemctl reload nginx
بعد تحديث الفرونت، افتح الموقع من نافذة Incognito أو استخدم Ctrl + Shift + R حتى تتجاوز كاش المتصفح.

7. تحديث WhatsApp Bridge

نفذ هذه الخطوات إذا كان التحديث يشمل ملفات داخل WhatsApp_Bridge:

cd /opt/clinic/ClinicProject/WhatsApp_Bridge
npm install --omit=dev
systemctl restart clinic-whatsapp

تأكد أن الخدمة تعمل:

systemctl status clinic-whatsapp

8. فحص Nginx والخدمات

nginx -t
systemctl reload nginx
curl -I http://167.86.119.12
curl -I http://167.86.119.12/api/Health

إذا لم يكن endpoint /api/Health متاحاً، افحص Swagger أو أي endpoint موجود:

curl -I http://167.86.119.12/swagger

9. مراقبة الأخطاء

إذا ظهرت مشكلة بعد التحديث، استخدم هذه الأوامر لمعرفة السبب:

journalctl -u clinic-api -n 120 --no-pager
journalctl -u clinic-whatsapp -n 120 --no-pager
journalctl -u nginx -n 120 --no-pager

وللمتابعة المباشرة أثناء التجربة:

journalctl -u clinic-api -f

مختصر التحديث اليومي

cd /opt/clinic/ClinicProject

cd Clinic_Booking

systemctl restart clinic-api

cd ../Clinic_Admin

cd ../WhatsApp_Bridge

systemctl restart clinic-whatsapp
