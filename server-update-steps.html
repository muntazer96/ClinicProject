فحص وحفظ التغييرات المحلية

قبل سحب أي تحديث من GitHub

4
فحص حالة Git
قبل السحب، افحص الملفات المعدلة على السيرفر.
git 
git status
لا تستخدم git reset --hard مباشرة إذا ظهرت تغييرات محلية — قد تحذف إعدادات مهمة كملف .env.production.
5
حفظ التغييرات المحلية في stash
احفظ كل التغييرات بما فيها الملفات غير المتتبعة مثل .env.production
git 
git stash push -u -m "server local changes before update"
تحقق 
git status
يجب أن تظهر: nothing to commit, working tree clean

سحب آخر تحديث من GitHub

جلب الكود الجديد من remote

6
git pull
git 
git pull origin message-version
إذا ظهر تعارض (conflict)، توقف فوراً ولا تكمل النشر. انسخ رسالة الخطأ وراجعها قبل تنفيذ أي أمر.

إرجاع ملف بيئة الفرونت

استعادة .env.production الخاص بالسيرفر

7
استعادة .env.production من stash
git 
git checkout stash@{0} -- Clinic_Admin/.env.production
فحص المحتوى 
nano Clinic_Admin/.env.production
المحتوى المتوقع 
VITE_API_BASE_URL=http://167.86.119.12/api
VITE_APP_VERSION=1.0.0
VITE_APP_BUILD_NUMBER=1
لا ترجع package-lock.json من الـ stash إلا إذا كنت متأكداً — الأفضل الاعتماد على النسخة القادمة من GitHub.

تحديث الفرونت Vue

Build & deploy Clinic_Admin

9
بناء ونشر الفرونت
bash 
cd /opt/clinic/ClinicProject/Clinic_Admin
npm install
npm run build
rm -rf /var/www/clinic-admin/*

systemctl reload nginx
بعد التحديث، افتح الموقع من نافذة Incognito أو اضغط Ctrl + Shift + R لتجاوز الكاش.

تنظيف الـ stash بعد النجاح

بعد التأكد من أن كل شيء يعمل

13
حذف الـ stash القديم
git 
git stash list
git stash drop stash@{0}
لا تحذف الـ stash إلا بعد التأكد أن ملف .env.production موجود وأن الموقع يعمل. إذا لم تكن متأكداً، اتركه — وجوده لا يضر.

مختصر التحديث اليومي

كل الأوامر مرتبة في مكان واحد

🚀 Full deployment cheatsheet
انسخ الكل وشغّله خطوة بخطوة
# ── 1. Git ──────────────────────────────────────
cd /opt/clinic/ClinicProject
git status
git stash push -u -m "server local changes before update"
git pull origin message-version
git checkout stash@{0} -- Clinic_Admin/.env.production

# ── 2. Backend .NET ─────────────────────────────
cd Clinic_Booking
dotnet publish /opt/clinic/ClinicProject/Clinic_Booking/Clinic_Booking.csproj \
  -c Release -o /var/www/clinic-api
systemctl restart clinic-api

# ── 3. Frontend Vue ─────────────────────────────
cd ../Clinic_Admin
npm install && npm run build
rm -rf /var/www/clinic-admin/*
cp -r dist/* /var/www/clinic-admin/
systemctl reload nginx

# ── 4. WhatsApp Bridge ──────────────────────────
cd ../WhatsApp_Bridge
npm install --omit=dev
systemctl restart clinic-whatsapp
